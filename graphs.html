Multimodal Graph Rendering (A/B Testing)

Accessibility • Screen-reader UX • Haptics + VoiceOver • Task success • Time-to-insight

A/B TestingWithin-Subjects Design Task-Based EvaluationQuant + Qual VoiceOverHaptics

Overview

Goal: determine which modality best supports nonvisual understanding of common charts. We compared VoiceOver only vs. VoiceOver + vibration across three chart types: scatter plots, bar charts, and dot charts.

Design

  • Phase 1 — Task performance (formative): participants completed interpretation tasks on each chart type in both modalities (counterbalanced).
  • Phase 2 — Knowledge test (summative): short-answer and multiple-choice items assessed retained understanding by chart type and modality.

Measures

  • Primary: task success, errors, time-to-insight
  • Secondary: confidence, perceived workload (short NASA-TLX), preference
  • Post-test: knowledge accuracy by modality and chart type

Key Findings

  • VoiceOver + vibration improved trend/outlier identification on scatter and dot charts, reducing time-to-insight versus VoiceOver only.
  • VoiceOver only remained efficient for bar charts with simple data series.
  • Participants reported higher confidence with the multimodal condition, especially for dense or noisy data.

Impact

  • Recommended modality-by-chart pairing guidelines (when to add haptics vs. when narration is sufficient).
  • Refined description grammar (title → axes → extrema/outliers → trend → takeaway) aligned to each modality.
  • Informed component patterns for accessible chart experiences used in follow-on projects.
